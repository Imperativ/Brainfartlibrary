\# Technical, Functional, and Legal Restrictions for the Prototype

Strictly observe the following \*\*technical, functional, and legal
limitations and requirements\*\* for this prototype:

\* \*\*No admin rights:\*\* You \*\*do not have administrative
rights\*\* in Confluence (version 9.2.4) or Jira (version 10.3.6 –
client/server version).

\* \*\*No access to the Storage Format Editor:\*\* You \*\*do not have
access to the Storage Format Editor\*\*.

\* \*\*Only standard graphical elements:\*\* You may \*\*only use native
graphical elements and functionalities\*\* that ConfiForms (version
3.17.7 by Vertuna.com), Confluence, or Jira make directly available to
\*\*standard users\*\* (e.g., “ConfiForms Field Definition”, “ConfiForms
IFTTT Integration Rules”, “Jira Issue Type”, “Confluence Macros”).

\* \*\*Do not reference outdated versions:\*\* The setup uses Confluence
(version 9.2.4), Jira (version 10.3.6 – client/server version), and
ConfiForms version 3.17.7. Always ensure that any information,
procedures, or terminology you provide are \*\*not based on outdated
versions\*\* of these products. Note that \*\*form field mapping\*\* may
differ significantly in formatting between versions.

\* \*\*Jira project:\*\* The Epic is \*\*always created in the Jira
project with the key BWPTLS24\*\*.

\* \*\*Fields for the prototype:\*\* For this initial prototype, only
the following Epic fields should be considered: \*\*Epic Name,
Description, Component/s, Priority, Due Date, and Reporter\*\*.

\* \*\*Confirmation after successful Epic creation:\*\* After
successfully creating an Epic, the user must be \*\*notified that a new
Epic has been created\*\*, including its \*\*name\*\* and \*\*issue
key\*\*. Ideally, a \*\*direct link\*\* to the Epic should also be
provided.

\* \*\*List of available fields and custom fields in Jira Epics:\*\* The
attached file \*Felder.txt\* contains all information displayed by the
ConfiForms JSON Mapping Helper.
