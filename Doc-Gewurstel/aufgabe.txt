Aufgabenstellung: Confiforms-Makro zur automatisierten Jira-Epic-Erstellung
Entwickle ein Confiforms-Makro für Confluence, das es Nutzer:innen ermöglicht, über eine Eingabemaske einen Namen für ein Jira-Epic anzugeben. Nach Klick auf „Okay“ soll im Jira-Projekt **BWPTLS24** automatisch ein Epic mit diesem Namen erstellt werden. Nach erfolgreicher Erstellung soll dem/der Nutzer:in ein interaktiver Link zum neuen Epic (Jira-Ticketnummer und Name) angezeigt werden. 
   - Bei einem ersten Prototypen sollen erstmal nur die Felder Epic-Name, Priorität, Fälligkeitsdatum und Ersteller berücksichtigt werden.
   - Das Epic wird immer im Jira-Projekt mit dem Key **BWPTLS24** erstellt.
   - Nach erfolgreicher Erstellung wird ein interaktiver Link zum neuen Epic angezeigt. Der Linktext enthält die Jira-Ticketnummer und den vergebenen Namen.
- Achte stets darauf, dass du keine Vorgehensweisen, Informationen oder Begifflichkeiten vermittelst, die eindeutig auf eine veraltete Version eines der Softwarekomponenten zurtück zu führen ist. Es wird Confluence 9.2.4 und Jira 10.3.6 (client/server version), sowie das confiforms plugin in version 3.17.7 von Vertuna.com benutzt.
- Benutze nur Elemente, die dir confiforms, confluence oder jira grafisch zur Verfügung stellt, also z.B. "Confiforms Field Definition" "Confiforms IFTTT Itegration Rules" usw. 
- Ich habe keine Adminrechte und kann auch nicht auf den Storage Format Editor zugreifen
oberstes Ziel ist es einen stabilen prototypen mit so wenig funktionalität wie möglich, aber so viel wie nötig zu erstellen.
